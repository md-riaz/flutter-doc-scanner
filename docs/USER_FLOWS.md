# User Flows

## 12) Main User Flows

### 12.1 Scan and Auto Upload

1. User logs in
2. User taps **Scan**
3. Camera opens
4. Document detected and captured
5. User reviews crop/enhancement
6. User adds more pages if needed
7. User selects project/folder
8. PDF is generated
9. Upload starts automatically
10. Status becomes uploaded / failed / pending

### 12.2 Scan Now, Upload Later

1. User logs in
2. User scans pages
3. PDF generated and stored locally
4. Status marked **pending**
5. Later, user opens upload queue
6. Retry/manual upload starts
7. Server confirms success
8. Status becomes **uploaded**

### 12.3 Document Management

1. User opens document list
2. Filters by project/status/date
3. Opens item
4. Renames / tags / categorizes / shares / downloads

### 12.4 Admin Monitoring

1. Admin logs into dashboard
2. Sees total uploads, failures, users
3. Filters by project/user/date/status
4. Resolves issues / checks logs / audits activity
